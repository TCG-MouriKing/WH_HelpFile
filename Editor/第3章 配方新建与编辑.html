# 第3章

软件主界面包含以下几个模块：

1）菜单栏；2）当前配方名；3）工具栏；4）检测项列表；5）图像视窗区；6）运行流程参数；7）软件当前配方路径；8）流程编辑窗口；9）流程编辑工具栏；10）工具属性；11）日志显示。

# 3.1 新建配方

# 3.1.1 新建

1）新建前准备：新建配方前需要将预处理算法库文件拷贝到软件运行目录“\CellGrid\Pres\Rear\Poly\”文件夹下。

2）主界面菜单栏中选中【文件】选项，在弹窗中选中【新建】选项打开配方新建窗口；

3）在新建配方窗口中设置配方名称，选择配方存储路径，并且根据当前配方使用工艺，选择工位、晶元类型、模板库与预处理库，完成后点击右上角的“√”进行保存。

![1715566664801](/image/第4章配方新建与编辑/1715566664801.png)

# 3.1.2图像加载

在图像设置区选择【图像】或【图像文件夹】按钮，通过获取图像文件或者图像文件夹遍历所有图像的方式加载图像。

在工具栏中选中【原图】显示彩图

# 3.1.3预处理执行

1）第一步：选中单击【预处理总览】按钮，切换出流程编辑窗口；

2）第二步：在左侧检测项列表中选中其中一个流程，软件会自动执行预处理算法，预处理参数区中显示经过预处理后输出的所有“图像集”、“区域集”、“数值集”。

3）检测出的缺陷加外接圆框出来方便快速定位

# 3.2 检测类、缺陷、流程新增与编辑

# 3.2.1新增检测类

1）软件工具栏中单击【新增检测类】按钮，在弹出的窗口中设置检测类名称，再引用设置预处理库提供的图像集、区域集、数值集。

2）新增完成后可在左侧配方流程列表看到新增的检测类名。

# 3.2.2新增检测缺陷

1）在软件左侧的检测列表中，将鼠标放置到对应的缺陷类上方，单击鼠标右键弹出选项窗口，单击选择【新建缺陷项】即可打开新增缺陷窗口，设置缺陷名及检测结果输出类型。

2）新增完成后，可在检测类列表中可看到新增的缺陷名。

# 3.2.3检测流程新增与编辑

# * 流程新增

1）在软件左侧的检测列表中，将鼠标放置到对应的缺陷项上方，单击鼠标右键弹出选项窗口，单击选择【新建流程】项即可打开新增流程窗口，设置缺陷流程名。

2）新增完成后，可在缺陷列表中看到新增的流程名。

# * 流程编辑

1）通过【预处理总览】按钮打开流程编辑窗口。

2）从流程编辑工具栏中拖工具到流程编辑窗口，流程编辑窗口中可根据设想的逻辑链接所有工具，实现检测流程，在编辑流程时选中工具，可在右侧的工具属性窗口查看与参数设置。

3）选中编辑器中的工具时，可在流程工具中滑动滚动条进行参数设置。

4）编辑完成之后 点击 保存流程 按钮 当流程文件发生变化时重置流程按钮生效，可以重置到配方保存之前的流程状态

# 工具复制粘贴

框选多个流程，在工具标题上右键菜单复制，在空白位置处右键粘贴

# 删除工具

框选多个流程，在工具标题上右键菜单删除，或CTRL+DEL组合键删除

# 特征窗体

特征窗体可以查看图像窗口中区域的几何特征

点击视窗中的缺陷区域，可在特征窗体中查看缺陷的详细几何特征

# 3.3 配方保存与编译、测试

# 3.3.1配方保存

选中菜单栏【文件】->【保存配方】按钮进行配方保存，保存完成后弹出提示“保存成功”，保存过程中严禁非法关闭程序以防止保存数据错误，破坏配方正常运行。

# 3.3.2配方编译

软件工具栏中选中【编译】按钮，在弹窗中设置dll名称及版本，导出文件在配方文件夹内。

# 3.3.3配方测试

1）在软件工具栏调试区域，点击相应的按钮，可进行配方测试。检测结果在左侧检测类列表反馈，每个缺陷流程如有缺陷输出，则会有不同颜色提示符显示，提示后可根据对应的流程进行调参优化。

# 3.4 检测类、缺陷、流程导入与导出

# 3.4.1检测类导入与导出

  1. 导入：从指定路径中将配置好的检测类导入到检测类列表中；

  2. 导出：在检测类上方右键单击弹出对话框，将选中的类导出到指定的路径；

# 3.4.2缺陷导入与导出

1.导入：在检测类上方右键单击弹出对话框，从指定路径中将配置好的缺陷导入到检测类列表中；

2.导出：在缺陷上方右键单击弹出对话框，将选中的缺陷导出到指定路径；

# 3.4.3流程导入与导出

  1. 导入 ：在缺陷上方右键单击弹出对话框，从指定路径中将配置好的流程导入到缺陷列表中；可以多选导入

  1. 导出 ：在流程上方右键单击弹出对话框，将选中的流程导出到指定路径；

# 3.5 旧版本配方转换新版本配方

由于软件更新，弃用了旧版本的配方，旧版本配方用新版本软件打开会报错无法使用，这时需要将旧配方转成新配方才可使用。以下为转换方法

1）启动新版配方编辑器打开配方，选择旧配方.JSON文件

2）打开旧配方后先加入一张图片进行测试，此时提示报错，暂时忽略，进行下一步

3）逐一点击打开缺陷类里的每一个流程，打开后有可能会报错，如下，点击确定忽略

4）此时会发现有些工具连线消失，需要重新手动连线

5）对照旧配方的连线，将新版本配方的连线重新手动连接。重新连线后保存流程

6）将所有流程都重新连线完成后，点击按钮运行。如果还会报错，如下

类似这种提示说明连线没有有些工具输入端完全连接，请仔细检查后再进行编译。

7）注意：最重要的一步：所有流程都必须点击展开查看一次，查看是为了让后台自动生成最新的代码，如果流程没有点开查看，

后台代码还是原来旧版本的代码，编译时还报各种异常错误。
